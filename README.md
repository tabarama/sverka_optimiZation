# sverka_optimiZation

Автоматизация «сверки» отчётов о продажах в Excel. Скрипт `sverka.py` заменяет
ручную последовательность действий (умная таблица + вычисляемый столбец + три
сводные таблицы) одним запуском.

## Что делает

Скрипт кладётся в папку с исходными `.xlsx` и при запуске для каждого файла
`X.xlsx` создаёт готовую книгу `сверка_X.xlsx`, а нетронутый оригинал переносит в
подпапку `old/`.

```
Было:                          Стало:
  sverka.py                      sverka.py
  Отчет по продажам.xlsx         сверка_Отчет по продажам.xlsx
                                 old/Отчет по продажам.xlsx
```

Для каждого файла воспроизводится ручной алгоритм:

1. **Комментарий.** Последний столбец отчёта, если у него пустой заголовок,
   получает название `Комментарий` (если уже назван — остаётся как есть).
2. **Умная таблица** `Таблица1` из области данных (заголовки + строки).
3. **Столбец `ЛПР+ЮЛ`** справа = `КВ ЛПР Итого` + `КВ ЮЛ Итого` (формат `#,##0.00`).
4. **Обновить всё** (пересчёт данных/связей).
5. **Три сводные таблицы** (из одного кэша по `Таблица1`):
   - **Партнёры/ЛПР** — строки: `Юридическое лицо`, `ID ТО`, `Комментарий`;
     значения: сумма `КВ ЮЛ Итого`, сумма `КВ ЛПР Итого`; фильтр `ЛПР+ЮЛ` (без `0,00`).
   - **Агенты** — строки: `Агент ЮЛ`; значения: сумма `КВ Агент Итого`.
   - **Операторы** — строки: `Агент ЮЛ`, `Оформил заявку`, `ID Оформителя`,
     `Юридическое лицо`, `Комментарий`; значения: сумма `КВ Оператор Итого`;
     фильтры: `КВ Оператор Итого` (без `0,00`) и `Оператор оплачено ранее` = `Нет`.
     Классический (табличный) макет; промежуточные итоги оставлены только у `Агент ЮЛ`.
6. **Автоподбор ширины** столбцов области сводных.
7. **Сохранение** в `сверка_X.xlsx`.

## Правила пропуска

Файл **не обрабатывается и не перемещается**, если:

- имя начинается с `сверка_` (это уже результат);
- это временный файл Excel (`~$...`);
- для него уже существует `сверка_X.xlsx` (повторно не пересоздаём);
- он лежит в подпапке `old/`.

Столбцы ищутся **по названию заголовка** (не по буквам столбцов), поэтому
перестановки колонок между отчётами допустимы. Если в файле нет одного из
обязательных столбцов (`ID ТО`, `Юридическое лицо`, `Агент ЮЛ`, `ID Оформителя`,
`Оформил заявку`, `КВ Оператор Итого`, `Оператор оплачено ранее`, `КВ Агент Итого`,
`КВ ЮЛ Итого`, `КВ ЛПР Итого`), файл пропускается с записью в лог, а оригинал
остаётся на месте.

## Требования

- **Windows** с установленным **Microsoft Excel** (скрипт управляет Excel через COM);
- **Python 3.9+**;
- пакет **pywin32**.

Настоящие сводные таблицы Excel (фильтры, классический макет, промежуточные итоги)
невозможно надёжно создать без самого Excel, поэтому скрипт работает только на
Windows с установленным Excel.

## Установка

```bat
pip install -r requirements.txt
```

Разово (ускоряет работу и включает раннее связывание COM):

```bat
python -m win32com.client.makepy "Microsoft Excel 16.0 Object Library"
```

## Запуск

Положите `sverka.py` в папку с исходными `.xlsx` и выполните:

```bat
python sverka.py
```

Ход работы и ошибки пишутся в консоль и в файл `sverka.log` рядом со скриптом.
Код возврата: `0` — все успешно, `1` — часть файлов с ошибкой, `2` — не удалось
запустить Excel.
